## 🚀 Coding Test Algorithm Study (Python)

### 이 저장소는 코딩 테스트 스터디의 알고리듬 구현 및 관련 문제 코드 리뷰를 위한 공간입니다.

### 동아리에서 다루고자 한 모든 알고리듬을 구현을 완료 후 관련 문제를 푸는 방식으로 진행합니다.

***

## 📅 운영방식

### 1. 알고리듬 구현 : 매주 정해진 알고리듬을 구현합니다.

### 2. 알고리듬과 관련된 문제 풀이 : 알고리듬 구현이 끝난 후 관련된 문제를 해결합니다.

### 2. 코드 업로드 : 각자 본인의 이름으로 브랜치를 만드신 후 금요일 오후 4시까지 main 브랜치로 PULL REQUEST(PR)을 보냅니다

### 3. 코드 리뷰 : 금요일 모임 당일, main 브랜치에서 취합된 코드를 기준으로 코드 리뷰 진행합니다.

***

## ✍️ 파일 네이밍 규칙

### 1. 파일명은 두가지 경우에 따라 달라집니다.

#### 1-1 자료주고 및 알고리듬 개녕 구현인 경우 : Concept_자료구조 or 알고리듬명.py

#### 1-2 관련된 문제 풀이인 경우 : 문제번호_레벌_문제명_자료구조 or 알고리듬명.py

### 2. 문제번호 확인: 프로그래머스 문제 URL 맨 뒤의 숫자 (예: .../lessons/12947)

### 3. 파일명 에시 : Concept_LinkdedList.py, 42576_Lv2_Athlete_LinkdedList.py 

### 4. 주의 : 혹시나 발생할 수 있는 인코딩 문제로 인한 한글 꺠짐 문제를 방지하기 위해 파일명은 영어로 작성합니다. 문제 출제하시는 분이 한글명과 영문명 두 버전으로 문제를 올려주세요

## 💬 커밋 메시지 규칙

### 일관된 커밋 기록을 위해 아래의 말머리(Prefix)를 사용해 주세요.

| 태그 | 의미 | 예시 |
| :--- | :--- | :--- |
| ✨ **DONE** | 문제를 완전히 풀었을 때 | `DONE: 12947_HarshadNumber` |
| 🏗️ **UNDONE** | 문제를 완전히 못 풀었을 때 | `UNDONE: 12947_HarshadNumber` |
| ⚙️ **Fix** | 코드 수정 및 리팩토링 | `Fix: 12947_HarshadNumber 효율성 개선` |
| 📝 **Docs** | README 등 문서 수정 | `Docs: README 파일 업로드` |
| 🚚 **Chore** | 폴더 구조 변경 등 기타 작업 | `Chore: 폴더 구조 변경` |
